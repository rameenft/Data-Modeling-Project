# VoiceInterface

## Overview

The VoiceInterface subsystem handles all audio input and output: converting user speech to text (STT), extracting affective features from audio, and synthesizing AI mediator responses back to speech (TTS). Voice is the primary interaction modality for the platform, chosen because it reduces friction for users navigating emotionally difficult experiences and enables richer affective signal capture.

## Key Concepts

- **Speech-to-text (STT)**: transcription of streaming audio to text. The platform uses a streaming STT system (Whisper or Deepgram) with partial transcripts available within 300ms of speech onset.
- **Voice activity detection (VAD)**: detects when the user is speaking vs. silent, triggering transcription and end-of-utterance detection. Critical for barge-in support.
- **Text-to-speech (TTS)**: neural voice synthesis converting the AI mediator's text responses to spoken audio. Uses SSML markup for prosody control calibrated to the user's detected emotional state (see [[AffectiveComputing]]).
- **Barge-in support**: the user can interrupt the AI mid-speech. VAD runs on the output audio channel; detected user speech pauses playback.
- **Dual-output transcription**: the system preserves both a raw transcript (with disfluencies, for affective feature extraction) and a cleaned transcript (for NLP downstream tasks).

## STT Pipeline

```
Microphone → WebSocket audio stream (16kHz mono)
  → VAD: filter silence, detect end-of-utterance
  → STT model: streaming transcription with partial results
  → Parallel: acoustic feature extraction for affect classification
  → Output: final transcript + word timestamps + confidence scores
```

Disfluencies (filler words, false starts, repetitions) are signal for [[AffectiveComputing]] and are preserved in the raw transcript before cleaning.

## TTS and Expressive Synthesis

The platform uses OpenAI TTS or ElevenLabs for neural voice synthesis. Prosody is controlled via SSML:
```xml
<speak>
  <prosody rate="slow" pitch="-2st">That sounds incredibly hard.</prosody>
  <break time="500ms"/>
  <prosody rate="medium" volume="soft">I'm here with you.</prosody>
</speak>
```

Prosody targets vary by conversation phase:

| Phase | Prosody |
|-------|---------|
| Greeting | Warm, medium pace, rising inflection |
| Listening/validation | Slow, soft, low pitch |
| Asking a question | Rising intonation, slight pause before |
| Closing | Warm, slower, falling contour |

The SSML markup is generated by the LLM alongside the response text, based on the current affective state — no separate prosody prediction module is needed.

## Latency Budget

Total perceived latency target: < 2.5 seconds (user stops speaking → AI response heard).

| Stage | Target |
|-------|--------|
| VAD end-of-speech | 200ms |
| STT transcription | 300ms |
| LLM first token | 500ms |
| TTS first audio chunk | 300ms |
| Audio delivery | 100ms |
| **Total** | **~1.4s** |

Streaming across all stages is mandatory — LLM generation and TTS synthesis must overlap with audio delivery.

## Privacy Design

Raw audio is never stored beyond the session. Only the cleaned transcript is retained (30-day maximum). On-device STT (Whisper tiny/base) is offered as a privacy-maximizing option for users who prefer no audio data leaving their device.

## Design Implications

- **Voice is a biometric**: audio data uniquely identifies users. Strict no-storage policy for raw audio is non-negotiable.
- **Multilingual support**: Whisper large-v3 supports 99 languages. Language detection runs before transcription to optimize model selection.
- **Empathetic voice persona**: the AI voice must convey warmth and stability, not corporate cheerfulness. Voice persona selection criteria: moderate warmth, gender-neutral, not too young, not authoritative.
- **CAPTCHA/bot defense**: the audio channel is a potential attack surface for automated abuse; VAD + turn-taking patterns help distinguish human users.
